## Overview

Toku sends webhook notifications for key events, allowing your systems to react in real-time to changes in grants, wallets, payroll, and more.

## Supported Webhook Sources

Toku integrates with several external services that deliver webhooks:

### Fireblocks (Custody)

Webhook endpoint: `POST /api/webhooks/fireblocks/transactions`

Receives notifications when custody transactions are created, confirmed, or fail. Used for:
- Token distribution confirmations
- Settlement transaction status updates
- Vault balance changes

### Safe (Multisig)

Webhook endpoint: `POST /api/webhooks/safe/transactions`

Receives notifications for multisig wallet events:
- Transaction proposals
- Signature confirmations
- Transaction execution status

### Persona (KYC)

Webhook endpoint: `POST /api/webhooks/persona`

Receives KYC verification status updates:
- Identity verification completed
- Verification approved/declined
- Document review status changes

### DocuSign (Document Signing)

Webhook endpoint: `POST /api/webhooks/docusign`

Receives document signing events:
- Envelope sent
- Recipient signed
- Envelope completed

## Setting Up Webhooks

To configure webhooks for your integration, contact the Toku team. Webhook endpoints require:

1. **Signature verification** — All webhooks include a signature header for verification
2. **HTTPS endpoint** — Your receiving endpoint must support HTTPS
3. **Idempotency** — Webhooks may be delivered more than once; use event IDs for deduplication

## Webhook Security

All webhook payloads include a cryptographic signature in the headers. Verify the signature before processing:

```javascript
const crypto = require('crypto');

function verifyWebhookSignature(payload, signature, secret) {
  const computed = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(computed)
  );
}
```

## Best Practices

- **Respond quickly** — Return a `200` within 5 seconds, then process asynchronously
- **Handle retries** — Failed deliveries are retried with exponential backoff
- **Log everything** — Store raw webhook payloads for debugging
- **Validate signatures** — Always verify the webhook signature before trusting the payload
