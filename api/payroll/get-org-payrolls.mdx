## Overview

Returns a list of all payroll runs for a specified organization, with optional filtering by year and state.

## Endpoint

```
GET /api/payroll/getPayrollByOrg
```

## Authentication

Requires `orgAdmin` role for the specified organization.

## Request

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `orgID` | number | Yes | The organization ID to retrieve payrolls for |
| `year` | number | No | Filter payrolls by year (e.g., 2024) |
| `state` | string | No | Filter by payroll state |

### Valid States

| State | Description |
|-------|-------------|
| `DRAFT` | Payroll created but not submitted |
| `PENDING_APPROVAL` | Awaiting approval |
| `APPROVED` | Approved, pending finalization |
| `FINALIZED` | Completed and processed |
| `CANCELLED` | Cancelled payroll run |

### Example Request

```bash
curl -X GET "https://api.toku.com/api/payroll/getPayrollByOrg" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "orgID": 100,
    "year": 2024,
    "state": "FINALIZED"
  }'
```

### JavaScript Example

```javascript
const response = await fetch('https://api.toku.com/api/payroll/getPayrollByOrg', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${process.env.TOKU_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    orgID: 100,
    year: 2024
  })
});

const payrolls = await response.json();
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `payrollID` | number | Unique payroll identifier |
| `orgID` | number | Organization ID |
| `payDate` | string | Payment date (ISO 8601) |
| `startDate` | string | Pay period start date |
| `endDate` | string | Pay period end date |
| `state` | string | Current payroll state |
| `type` | string | Payroll type |
| `employeeCount` | number | Number of employees in payroll |
| `totalGross` | number | Total gross pay amount |
| `createdAt` | string | Creation timestamp |

### Error Responses

| Status | Code | Description |
|--------|------|-------------|
| 401 | `UNAUTHORIZED` | Invalid or missing API key |
| 403 | `FORBIDDEN` | User lacks permission for this organization |
