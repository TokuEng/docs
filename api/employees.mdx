## Overview

The Employees API lets you manage user roles within your organization. Users are identified by their `roleInOrgID`, which represents a specific person's role in a specific organization. Each user can have different roles across different organizations.

---

## Get User Role in Org

Retrieves the role details for a specific user in the organization.

```
GET /api/tokuApi/v1/getUserRoleInOrg
```

**Query Parameters:**
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `roleInOrgID` | uuid | No | Internal role identifier |
| `externalEmployeeID` | string | No | External HRIS identifier |

At least one identifier must be provided.

**Example:**
```bash
curl "https://app.toku.com/api/tokuApi/v1/getUserRoleInOrg?externalEmployeeID=EMP-001" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "x-role-type: CLIENT_ORG_ADMIN"
```

## Create User Role in Org

Creates a new user and assigns them a role in your organization. Use this to onboard employees programmatically.

```
POST /api/tokuApi/v1/createUserRoleInOrg
```

**Request Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | string | Yes | User's email address |
| `givenName` | string | Yes | First name |
| `familyName` | string | Yes | Last name |
| `role` | string | Yes | Role to assign (e.g., `RECIPIENT`, `CLIENT_ORG_ADMIN`) |
| `externalEmployeeID` | string | No | External HRIS identifier |
| `payrollEmployeeID` | string | No | Payroll system identifier |

**Example:**
```bash
curl -X POST https://app.toku.com/api/tokuApi/v1/createUserRoleInOrg \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "x-role-type: CLIENT_ORG_ADMIN" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "jane.smith@company.com",
    "givenName": "Jane",
    "familyName": "Smith",
    "role": "RECIPIENT",
    "externalEmployeeID": "EMP-001"
  }'
```

**Response:** The created role-in-org object with `roleInOrgID`.

## Update User Role in Org

Updates an existing user role (e.g., change external ID, update name).

```
PUT /api/tokuApi/v1/updateUserRoleInOrg
```

**Request Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `roleInOrgID` | uuid | Yes | The role to update |
| `externalEmployeeID` | string | No | Updated external ID |
| `payrollEmployeeID` | string | No | Updated payroll ID |
| `givenName` | string | No | Updated first name |
| `familyName` | string | No | Updated last name |

**Example:**
```bash
curl -X PUT https://app.toku.com/api/tokuApi/v1/updateUserRoleInOrg \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "x-role-type: CLIENT_ORG_ADMIN" \
  -H "Content-Type: application/json" \
  -d '{
    "roleInOrgID": "role-uuid",
    "externalEmployeeID": "EMP-001-UPDATED"
  }'
```

---

## Batch Update Roles in Org

Updates multiple user roles in a single request.

```
PUT /api/tokuApi/v1/batchUpdateRoleInOrg
```

**Request Body:** Array of update objects, each containing `roleInOrgID` and the fields to update.

**Example:**
```bash
curl -X PUT https://app.toku.com/api/tokuApi/v1/batchUpdateRoleInOrg \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "x-role-type: CLIENT_ORG_ADMIN" \
  -H "Content-Type: application/json" \
  -d '[
    {"roleInOrgID": "role-uuid-1", "externalEmployeeID": "EMP-001"},
    {"roleInOrgID": "role-uuid-2", "externalEmployeeID": "EMP-002"}
  ]'
```

## Get User Product Types

Retrieves the product types available for users in the organization.

```
POST /api/tokuApi/v1/getUserProductTypes
```

## Available Roles

| Role | Description |
|------|-------------|
| `RECIPIENT` | Grant recipient / employee — can view their grants and manage their wallet |
| `CLIENT_ORG_ADMIN` | Full administrative access to the organization |
| `PAYROLL_ADMIN` | Access to payroll operations and settlement |
| `FINANCE_ADMIN` | Access to financial reporting |
| `INVESTOR` | Read access to investments and warrants |
| `MANAGER` | Manager-level access for team operations |
